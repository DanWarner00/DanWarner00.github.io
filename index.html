P.01SHIPPED
AI Job Search Platform
What started as an excuse to start learning the new OpenClaw gateway turned into a very useful project for assisting in the job search and application process.
PythonFlaskClaude API PlaywrightSQLiteTailwindCSS
GitHub

Upload your resume, set preferences for location, job type, salary range, and keywords, and the platform scrapes leading job boards like Indeed and Adzuna for relevant listings. Connected to the Claude API with a custom system prompt, it grades each job with a match score and stores everything in a SQLite database to avoid duplicate entries. Features include a Kanban board for tracking jobs you've applied to, scheduled interviews, rejections, and inactive listings — giving you a clear picture of your search at a glance. A cover letter generator pulls from your resume and the preferences you've provided to produce a tailored letter based on each job's description. There's also a calendar for scheduled interviews and an AI summarizer that explains why you are (or aren't) a good fit for the position.

The match summarizer has been useful for flagging jobs where I clearly don't meet the requirements and for affirming when something looks like a strong fit. The Kanban keeps the application pipeline organized at a glance. Down the road, I'd like the system to learn from jobs I've marked as "not relevant" to further refine what gets scraped. Note: application statuses shown in the Kanban are not reflective of my actual job search progress — they're there to demonstrate the feature.

What it does
Preference-based scraping across Indeed and Adzuna
AI match scoring and fit explanation per listing
Cover letter generator using resume + job details
Kanban board: Applied → Interview → Offer → Rejected
Interview calendar and analytics dashboard

P.03DEMO READY
KUMU — Dental SOAP Documentation AI
Turns a messy 10-minute dentist conversation into a clean clinical note. All local — no patient data leaves the building.
PythonLangGraphPydantic Local LLMsFlaskGradio
GitHub

KUMU runs a raw dentist-patient transcript through a multi-node LangGraph pipeline — one node per SOAP section (Subjective, Objective, Assessment, Plan, Personal). Pydantic output validation at every step means the clinical note comes out in a usable format rather than a wall of AI text.

Built to run on local models so patient data never leaves the clinic. Includes both a Flask web UI and a Gradio prototype. Drop in a transcript, get back a formatted clinical note ready for the patient record.

How it works
Multi-node LangGraph graph — one node per SOAP section
Pydantic models enforce structured output at every step
Runs fully local — no patient data sent to external APIs
Flask web app + Gradio prototype
System prompts tuned for dental clinical language
